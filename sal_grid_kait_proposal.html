SECTION 01

SAL Grid 방법론 개요

3차원 좌표계 기반 프로젝트 관리 방법론의 핵심 개념을 소개합니다.

SAL Grid란?

SAL Grid는 3차원 좌표계 기반 프로젝트 관리 방법론입니다. 모든 작업이 3차원 좌표(S, A, L)로 고유하게 식별됩니다.

Stage (S) 프로젝트의 시간축. 단계별 진행 (S1 → S2 → ... → Sn)
Area (A) 작업 영역. 독립적 도메인 구분 (DO, PH, VD, OR 등)
Level (L) 작업 깊이. 같은 Stage×Area 안에서의 세부 순서

SAL ID 구조

형식: S{stage}{AREA}{level}{variant?}

S3DO1 3 (분류/정리) DO (일반문서) 1 일반문서 분류 첫 작업
S4PH2 4 (디지털화) PH (사진) 2 사진 디지털화 두 번째 작업

핵심 특징 5가지

1

No-Code 정의

코드 없이 프로젝트 구조 정의. 누구나 SAL ID만으로 작업 정의 가능

2

자기설명적 ID

ID 자체가 작업의 위치/성격/순서를 설명. S4PH2 = "디지털화 단계의 사진 영역 두 번째 작업"

3

자동 의존성

Stage 순서로 의존성이 자동 결정. S3 완료 → S4 시작. 수동 의존성 설정 불필요

4

Stage Gate 검증

각 Stage 경계에서 품질 검증. 미완료 작업이 있으면 다음 Stage 진입 불가

5

ID Chain 추적

SAL ID로 작업 간 관계 추적. 어떤 작업이 어떤 작업에 의존하는지 즉시 파악

SECTION 03

Before & After 비교

SAL Grid 도입 전후의 프로젝트 관리 방식을 비교합니다.

BEFORE — 현재 방식 (추정)
  1. 엑셀 기반 관리: 프로젝트마다 별도 엑셀 파일. 버전 충돌, 동시 편집 불가
  2. 구두/메신저 커뮤니케이션: 작업 상태 확인을 위해 담당자에게 직접 물어야 함
  3. 경험 의존적 분류: 기록물 분류가 담당자 경험에 의존. 신규 인력 투입 시 품질 저하
  4. 수동 진행률 추적: 전체 프로젝트 진행률을 파악하려면 여러 시트를 수동 취합
  5. 인수인계 어려움: 담당자 변경 시 업무 파악에 며칠 소요. 암묵지 의존
  6. 재작업 빈발: 이전 단계 미완료 상태에서 다음 단계 진행 → 재작업 발생
AFTER — SAL Grid 적용 후
  1. 구조화된 SAL Grid: 하나의 Grid로 전체 프로젝트 관리. 실시간 동기화
  2. SAL ID 즉시 확인: S3DO1 상태? → Viewer에서 즉시 확인. 물어볼 필요 없음
  3. 체계적 분류 코드: Area 코드(DO/PH/VD/OR)로 분류 기준 표준화. 신규 인력도 즉시 적용
  4. 자동 진행률 산출: Stage별 완료율 자동 계산. 대시보드에서 한눈에 파악
  5. 자기설명적 구조: SAL ID만 보면 업무 파악 완료. 인수인계 시간: 며칠 → 수분
  6. Stage Gate 차단: S3 미완료 → S4 진입 불가. 재작업 원천 차단

SECTION 05

SAL Grid + AI Platoon: 대량 AI 병렬 투입

AI 에이전트를 군사 소대 편제 방식으로 조직하여 기록관리를 혁신하는 방법론입니다.

5-1. Platoon Formation이란?

AI 에이전트를 군사 소대(Platoon) 편제 방식으로 조직하여 42명의 AI가 동시에 작업하는 혁신적 방법론입니다. 소프트웨어 개발뿐 아니라, 기록관리처럼 대량의 반복·분류·검수 작업이 필요한 분야에서 극대화된 효과를 발휘합니다.

핵심 개념: 군사 편제를 AI에 적용

  • 소대장 (Opus) — 전체 작전을 지휘하는 오케스트레이터. 직접 작업하지 않고 임무를 분해하여 분대에 배정합니다.
  • 3개 분대 (Alpha / Bravo / Charlie) — 각 분대장(Sonnet) 1명 + 정규병(SubAgent) 11명 = 12명. 분대장이 임무를 세분화하여 정규병에게 병렬 투입합니다.
  • 정규병 11명 (분대 공통) — frontend-dev, backend-dev, database-dev, ui-designer, api-dev, code-reviewer, test-runner, debugger, security-spec, doc-writer, devops-ts. 작업 성격에 맞춰 투입됩니다.
  • 용병 풀 (4명 공유) — Codex(코드 생성), Gemini(대용량 분석), Grok(실시간 검색), Perplexity(리서치). 소대장/분대장 누구나 호출 가능한 공유 자산입니다.
  • 18개 전문 스킬 — 방법론(SAL Grid), 통제(Review), 배포, 이미지생성, 문서생성, 보안감사, 테스트 등 11개 Tier의 전문 스킬을 장착합니다.

작전 수행 절차 (9단계)

1. 팀 생성 → 2. 분대장 3명 스폰 → 3. 용병 연동 → 4. 스킬 장착 → 5. 대기 보고 → 6. 작전 승인 → 7. 임무 수행 → 8. 결과 보고 → 9. 해산

나머지는 소대장이 자율적으로 판단하여 42명을 지휘합니다.

5-3. 벌떼 전략: 기록물 1만건을 42명이 동시 처리

기존 방식
  • 담당자 5명이 순차적으로 작업
  • 메타데이터 입력: 1인당 2,000건 × 5분 = 10,000분(167시간)
  • 전체 소요: 833시간 (약 104일)
Platoon 방식
  • 3개 분대 × 11명 = 33명의 AI가 동시 투입
  • Alpha: 문서 3,000건 메타데이터 자동 추출
  • Bravo: 사진 4,000건 자동 분류 + 태깅
  • Charlie: 나머지 3,000건 + 품질검수 병렬 수행
  • 전체 소요: 약 25시간 (3일) — 속도 향상: 33배

5-4. SAL Grid × Platoon 시너지

SAL Grid (구조) + Platoon (실행) = 결과
무엇을 할지 정의 + 누가 할지 배정 = 구조화된 대량 병렬 처리
Stage Gate 품질 관리 + 분대별 검증 = 다중 검증으로 품질 향상
자동 의존성 + 병렬 작업 식별 = 최적 스케줄링
SAL ID 추적 + 42명 작업 통합 = 완벽한 작업 추적

"SAL Grid가 작업의 '지도'라면, Platoon Formation은 그 지도 위를 달리는 '군단'입니다."

구조(SAL Grid) × 실행력(AI Platoon) = 기록관리 혁신
