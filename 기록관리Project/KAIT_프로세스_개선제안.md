## 2. 사업 영역 및 업무 내용

### 2-1. 핵심 비즈니스 흐름

```
현황파악·체계수립  →  기록수집·정리  →  시스템 등록·관리  →  전시·편찬·콘텐츠 개발
    [컨설팅]            [수집/정리]          [솔루션]             [콘텐츠개발]
```

---

### 2-2. 기록관리 컨설팅

6가지 핵심 업무 영역:

1. **기록관리 체계 수립** — 조직·프로세스·인프라·시스템 전반 컨설팅
2. **기록 수집** — 수집정책 수립, 리드파일 개발, 수집교육, 내외부 수집, 이벤트 수집
3. **분류체계 개발** — 조직·직제·업무분석을 통한 분류체계 및 기록관리기준 수립
4. **기록 정리** — 실물정리, DB구축, 디지털화, 수장고 구축, 보존용품 개발
5. **시스템 구축** — 국내외 표준 준용 시스템 구축 컨설팅
6. **기업아카이브 관리** — 사료관리 컨설팅, 수집, 서고관리, 콘텐츠 개발, 시스템 구축

주요 고객사: SK, GS그룹, 현대자동차, 국가기록원, 서울기록원, 국립중앙박물관, 한양대학교 등

---

### 2-3. 시스템 구축 — 감어인 솔루션 5종

| 솔루션 | 대상 | 주요 기능 |
|--------|------|-----------|
| **감어인R** | 현용기록관리 | 등록/정리/이관/인수, 분류/보존/평가, 열람/대출/검색 |
| **감어인M** | 아카이브(수집기록) | 수집, 등록·분류, 보존, 활용·검색, 연혁·구술, 통계 |
| **감어인P** | 사진관리 | 사진 기록 전문 관리 |
| **감어인O** | 구술기록관리 | 구술채록 산출물 관리 |
| **감어인I** | 보유목록관리 | 비전자기록 목록 관리 |

**감어인R 특징:**
- 생산시스템 실시간 연계 → 이관 비용 0원
- 전자/비전자/수집/구술/특수유형 기록물 통합관리
- 다중분류(업무/출처/주제/시대) 지원
- 도입 기관: 청와대, 국회, 한국은행, 예금보험공사, 인천항만공사 등

**감어인M 특징:**
- 2001년 국내 최초 개발, 현재 감어인M5
- ISAD(G), ISAAR(CPF) 국제기준 준수
- 수집기록·역사기록·생산기록 통합관리

### 2-4. 수집 및 정리

**기록 수집 (단계별):**
1. 수집업무매뉴얼 개발
2. 현장 수집활동 및 구술채록
3. 수집 안내 홍보, 내부 교육

**기록 정리 및 보존 (단계별):**
1. 실물기록 정리
2. 스캐닝 및 사진 촬영 (디지털화)
3. 보존용품 적용 및 서가 배치
4. 수장고 구축

---

### 2-5. 콘텐츠 개발

5가지 콘텐츠 유형:

1. **디지털 아카이브 구축** — 타임라인, 위치기반, 온라인 전시 포함 서비스 플랫폼
2. **주제별 기록콘텐츠 개발** — 스토리 집필, 기록 선별, 삽화·구술 보완
3. **온·오프라인 전시** — 기관역사·특정주제 전시, VR 디지털화
4. **편찬** — 역사서, 도록, 백서 발행
5. **역사·인터뷰 영상 제작** — 기관/기업 역사 영상, 주요 인물 인터뷰

**프로세스:** 분석 → 기획 → 콘텐츠 개발 → 서비스 제공

---

### 2-6. 보존서고용품

- **기록관 비상용품 키트** — 대형(54종), 소형(34종)
- **서고 소독** — 천연약재 추출물 기반 친환경, ULV 시스템, 살균·살충 99% 이상
- **정리/서고용품** — 보존봉투(중성지), 보존상자(표준규격)

## 3. 현재 프로세스의 한계

| 사업 영역 | 현재 방식 | 주요 한계 |
|-----------|-----------|-----------|
| 기록관리 컨설팅 | 현장 분석 → 수작업 체계 수립 | 컨설턴트 역량 의존도 高, 표준화 어려움 |
| 시스템 구축 | 설치형 솔루션, 5종 별도 운영 | 시스템 간 사일로, 클라우드 전환 미흡 (업데이트 지연·비용 重複) |
| 수집 및 정리 | 현장 수집 → 스캐닝 → 수작업 DB 입력 | 수작업 비중 높음, 메타데이터 품질 편차 |
| 콘텐츠 개발 | 기획→집필→제작 순차 수행 | 긴 납기, 재활용성 낮음 |
| 보존서고용품 | 제품 공급 중심 | 서비스와 분리된 단품 사업 |

## 4. 변화 동인 (Why Now?)

기록관리 산업을 둘러싼 외부 압력이 빠르게 가중되고 있다.

| 동인 | 내용 |
|------|------|
| **AI 기술 대중화** | LLM·OCR·STT 성능이 실무 적용 수준에 도달, 경쟁사 도입 가속 |
| **클라우드 전환 의무화** | 정부의 공공 클라우드 우선 정책(Cloud First) 확대로 설치형 솔루션 수요 감소 |
| **기록관리 법·제도 강화** | 공공기록물법 개정 논의, 기업 ESG 관점 아카이브 의무화 논의 증가 |
| **고객 예산 압박** | 프로젝트 단위 대형 구축 예산 확보 어려움 → 소규모 구독형 수요 증가 |

## 5. AI 기반 프로세스 개선 제안

> 제안은 실행 시점(단기→중기→장기) 순으로 배열합니다.

---

### [단기] 제안 1. AI 기반 기록물 자동 분류·메타데이터 추출

**현재 문제:**
수집된 실물/스캔 기록물을 담당자가 수작업으로 분류하고 메타데이터(생산일자·생산자·주제·분류번호)를 입력. 품질 편차가 크고 속도가 느림.

**개선 방안:**
- OCR + LLM 결합으로 스캔 문서에서 생산일자·생산기관·주제어 자동 추출
- 기존 감어인 시스템의 분류체계를 학습 데이터로 활용, 자동 분류코드 추천
- 담당자는 AI 추천 결과를 **검토·승인**하는 역할로 전환 (Review-in-the-Loop: AI가 추천하고 담당자가 최종 판단하는 방식)
- 추진 방식: 외부 OCR/LLM API 활용 + 감어인 시스템 플러그인 방식으로 내재화

**기대 효과:** 정리 업무 속도 3~5배 향상, 메타데이터 품질 균일화

> 난이도: ★★☆ / 투자비용: 中 / 기대 ROI: 高 (인건비 절감 직결)

---

### [단기] 제안 2. 구술기록 AI 자동 전사(Transcription) + 요약

**현재 문제:**
구술채록은 인터뷰 진행 → 녹음/녹화 → 수작업 전사 → 편집의 흐름. 전사 작업이 시간·비용 집약적.

**개선 방안:**
- AI 음성인식(STT)으로 구술 인터뷰 자동 전사 (Whisper 등 오픈소스 모델 활용 가능)
- LLM으로 핵심 내용 요약, 키워드 태깅, 관련 기록물 연결 추천
- 감어인O(구술기록관리시스템)에 AI 전사 기능 내재화
- 추진 방식: 검증된 STT API(클로바 노트 등) 연동으로 빠르게 적용

**기대 효과:** 전사 비용 70% 이상 절감, 구술기록 활용 범위 확대

> 난이도: ★☆☆ / 투자비용: 低 / 기대 ROI: 高 (즉시 원가 절감 효과)

---

### [단기] 제안 3. 컨설팅 지식베이스 및 표준 방법론 플랫폼화

**현재 문제:**
26년간 437개 고객사, 2,526개 프로젝트의 경험이 컨설턴트 개인에게 체화 → 지식 이전 어려움, 신규 컨설턴트 육성에 시간 소요.

**개선 방안:**
- 과거 프로젝트 산출물(분류체계, 기록관리규정, ISP보고서)을 **지식베이스 DB**로 구축
- RAG(검색증강생성) 기반 내부 AI 어시스턴트 도입: "○○유형 기관의 분류체계 사례" 즉시 검색
- 컨설팅 방법론을 단계별 체크리스트·템플릿으로 표준화
- 추진 방식: 자체 내부 프로젝트로 구축 (초기 투자 최소화)

**기대 효과:** 컨설팅 품질 균일화, 신규 인력 온보딩 속도 향상, 암묵지의 형식지화

> 난이도: ★★☆ / 투자비용: 低中 / 기대 ROI: 高 (인재 의존 리스크 해소)

---

### [중기] 제안 4. 감어인 플랫폼 클라우드 SaaS 전환

**현재 문제:**
감어인R/M/P/O/I가 5종 개별 설치형 솔루션으로 운영. 고객사마다 별도 구축 → 유지보수 비용 高, 업데이트 배포 복잡.

**개선 방안:**
- 5개 솔루션을 클라우드 단일 플랫폼(멀티 테넌트)으로 통합
- 소규모 기관(대학, 재단, 종교기관)에는 **구독형 SaaS** 모델 제공 → 초기 구축 비용 없이 월정액 도입
- 대형 공공기관은 온프레미스/하이브리드 유지
- 공공기관 대상: **CSAP(클라우드 보안 인증)** 취득 및 행안부 공공 클라우드 지침 준수를 선결 조건으로 추진
- 추진 방식: 전문 클라우드 파트너(AWS/NCP)와 공동 개발

**기대 효과:** 中소형 고객사 저변 확대, 반복 매출(MRR) 구조 확보, 유지보수 효율화

> 난이도: ★★★ / 투자비용: 高 / 기대 ROI: 高 (장기 MRR 확보로 사업 안정성)

---

### [중기] 제안 5. 디지털 아카이브 대국민 서비스 포털 표준화

**현재 문제:**
콘텐츠 개발(디지털 아카이브, 온라인 전시 등)이 고객사별 개별 구축 → 비용 높고 재사용성 낮음.

**개선 방안:**
- **표준 포털 템플릿** 개발: 타임라인, 위치기반 지도, 인물 DB, 온라인 전시관을 모듈화
- 고객사는 템플릿 선택 후 콘텐츠만 입력 → 구축 기간·비용 대폭 단축
- 감어인M과 API 자동 연동으로 시스템↔서비스 포털 이중 입력 제거
- 추진 방식: 자체 개발 후 고객사별 화이트라벨링(White-labeling) 제공

**기대 효과:** 콘텐츠 구축 납기 50% 단축, 고객 만족도 향상

> 난이도: ★★☆ / 투자비용: 中 / 기대 ROI: 中高 (프로젝트 마진율 개선)

---

### [장기] 제안 6. 보존서고용품 IoT 연계 스마트 서고 서비스

**현재 문제:**
보존서고용품 사업이 단순 물품 공급에 머물러 있어 부가가치가 낮음.

**개선 방안:**
- 온·습도 센서 + IoT 모니터링 솔루션을 용품과 결합해 **스마트 서고 패키지**로 업그레이드
- 감어인 시스템과 연동하여 이상 환경 감지 시 자동 알림
- 보존 상태 데이터를 누적해 예방적 보존처리 시기 추천
- 추진 방식: IoT 전문 파트너사와 협업하여 하드웨어-소프트웨어 번들 상품 구성

**기대 효과:** 용품 사업의 서비스화(servitization), 고객 락인(lock-in) 강화

> 난이도: ★★★ / 투자비용: 中高 / 기대 ROI: 中 (신규 수익원 개척)

## 7. 핵심 요약

KAIT는 26년 경험의 탄탄한 도메인 전문성을 갖고 있지만, **수작업 중심·설치형 솔루션·프로젝트 단위 수익** 구조에 머물러 있습니다.

핵심 개선 방향 3축:

1. **AI 자동화** — 반복 수작업(분류, 메타데이터 입력, 전사)을 AI로 대체
2. **SaaS 전환** — 설치형 → 클라우드 구독형으로 반복 매출 구조화
3. **지식베이스 플랫폼화** — 축적된 26년 경험을 조직 자산으로 형식지화

이 세 축을 중심으로 전환하면 매출 구조 개선과 경쟁력 강화를 동시에 달성할 수 있습니다.
